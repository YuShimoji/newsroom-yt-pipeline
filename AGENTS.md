# Agent Instructions

Project-local instructions come first. Global rules are fallback only.

Use this file as a thin entry point only. Do not grow it into procedures,
status, roadmap, closeout templates, or history.

Read restart authority in this order:

1. `docs/HANDOFF.md`
2. `docs/RUNTIME_STATE.md`
3. `docs/DEVELOPMENT_PRACTICES.md`
4. `artifacts/ARTIFACTS.md` when reviewable artifacts are involved.
5. `docs/PROJECT_SPEC.md` only when the specification boundary is needed.
6. `docs/META_REVIEW_LEDGER.md` only when blocker or scope-review context is needed.

If a referenced file is missing, treat the reference as stale, not as a blocker.

For residual work, report purpose, effect, requirements, state, owner, and next move.
